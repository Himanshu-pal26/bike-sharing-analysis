Bike-Sharing Demand Analysis

Tools Used:Excel | SQL (MySQL) | Power BI  

This project analyzes bike-sharing ride data to understand user behavior, peak demand hours, and high-traffic locations.

Project Overview
The goal of this project is to analyze bike-sharing data and identify patterns that can help improve operational efficiency and customer experience.


 Data Cleaning & Preparation
- Removed duplicate and blank records  
- Cleaned and formatted date and time columns  
- Created new calculated columns such as weekday and ride duration  
- Structured data for analysis and visualization  


Analysis Performed
- Member vs Casual rider comparison  
- Day-wise demand analysis  
- Peak hour analysis  
- Top 5 most used stations  


Tools Used
- Excel:Data cleaning, pivot tables, charts  
- SQL (MySQL): Data querying and aggregation  
- Power BI: Interactive dashboard and visualizations  


Key Insights
- Identified peak usage hours of the day  
- Found which weekdays have highest ride demand  
- Compared usage between members and casual riders  
- Identified high-traffic bike stations  


This project demonstrates an end-to-end data analytics workflow from raw data to business insights.
